# Interactive API reference (Qeet ID) — how it works

**Status: implemented.** The Qeet ID API reference at `/id/api-reference` is an interactive
reference generated from the vendored OpenAPI specs, with request/response schemas and a
"Try it" console (powered by `fumadocs-openapi@11`).

## Pieces

- **Vendored specs**: `content/specs/qeet-id/{auth,management,federation,developer,operations}.yaml`
  — byte-faithful copies of `../qeet-id/api/openapi/*.yaml`, except a production server
  (`https://api.id.qeet.in`) is prepended to each `servers:` list so the "Try it" console
  targets prod. (`.prettierignore` covers `content/specs/` + the generated output.)
- **Generation**: `scripts/generate-openapi.mjs` runs `createOpenAPI` + `generateFiles`
  (`per: "tag"`) once per spec → `content/docs/id/api-reference/<spec>/*.mdx` (+ `meta.json`).
  Wired as `generate:openapi`, run automatically by `predev` / `prebuild`.
- **Server registry**: `src/lib/openapi.ts` — `createOpenAPI({ input: [...5 specs...] })`. The
  `input` paths MUST match the `_openapi.preload` keys in the generated MDX (leading `./`).
- **Client component**: `src/components/api-page.tsx` — `createOpenAPIPage()` (it's a
  client-only factory, hence the `"use client"` module).
- **Renderer wiring**: `src/app/(docs)/[...slug]/page.tsx` — for pages whose frontmatter has
  `_openapi`, it `await openapi.preloadOpenAPIPage(page)` and injects the bundled schema into
  `<APIPage>` via the `preloaded` prop (the client component can't read files itself).
- **Loader plugin**: `openapiPlugin()` in `src/lib/source.ts` (decorates the nav with HTTP
  method/webhook labels). **CSS**: `@import "fumadocs-openapi/css/preset.css"` in `global.css`.
- **Nav**: `content/docs/id/api-reference/{index.mdx,meta.json}`; `/id/api` was retitled to
  "API conventions" (auth schemes, errors, pagination) and its hand-curated endpoint pages
  (users/tenants/roles/sessions) were removed in favor of the generated reference.

## Regenerating when the qeet-id API changes

```bash
cd qeet-docs && nvm use node
cp ../qeet-id/api/openapi/{auth,management,federation,developer,operations}.yaml content/specs/qeet-id/
# re-add the production server line to each spec's servers: list (see content/specs/qeet-id/README.md)
pnpm generate:openapi      # also runs automatically on predev / prebuild
pnpm typecheck && pnpm build
```

The drift between the vendored copies and qeet-id is what the workspace `release-coordinator`
+ `api-reference-engineer` agents watch.

## Notes / gotchas

- Requires `fumadocs-core`/`fumadocs-ui` `>= 16.10` (bumped from 16.9.3) for `fumadocs-openapi@11`.
- The generated MDX under `content/docs/id/api-reference/**` is regenerated by `prebuild` —
  don't hand-edit it. `.prettierignore` excludes it from the format gate.
- Stale singular references to `../qeet-id/api/openapi.yaml` remain in
  `api-reference-engineer.md` / `release-coordinator.md` — fold them to the split
  `api/openapi/*.yaml` paths in a future pass.
