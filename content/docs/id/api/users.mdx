The Users API manages user identities. Authenticate server-side with an API key
(`Authorization: ApiKey qk_…`).

## Create a user

<APIEndpoint method="POST" path="/v1/users" title="Create a user" />

```bash title="curl"
curl -X POST https://api.id.qeet.in/v1/users \
  -H "Authorization: ApiKey $QEETID_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "email": "alex@acme.com", "display_name": "Alex Chen" }'
```

Email is **globally unique**; a duplicate returns `409`.

## Soft-delete & recycle bin

Deleting a user is a **soft-delete** — the user is disabled and lands in a recycle bin
you can list, then either **restore** or **purge** (irreversible).

<APIEndpoint method="DELETE" path="/v1/users/{id}" title="Soft-delete a user" />
<APIEndpoint
  method="GET"
  path="/v1/users/deleted"
  title="List soft-deleted users"
/>
<APIEndpoint
  method="POST"
  path="/v1/users/{id}/restore"
  title="Restore a user"
/>
<APIEndpoint
  method="POST"
  path="/v1/users/{id}/purge"
  title="Permanently purge a user"
/>

<Callout type="warn" title="Purge is irreversible">
  Purge (and [GDPR erasure](/id/platform/gdpr)) permanently removes personal
  data. The audit record of the deletion survives with PII redacted.
</Callout>
