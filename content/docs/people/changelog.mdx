Qeet People is **in design**. There's no public API yet; this page will track shipped
milestones once development begins. For now it records design milestones.

<Callout type="info" title="In design">
  Qeet People is documented from its product requirements. The repo is a
  skeleton — no endpoints or SDKs exist yet. See [Concepts](/people/concepts)
  for the planned model and the [roadmap](/people/roadmap) for the phased plan.
</Callout>

## Design phase

- **Product requirements complete** — personas, feature modules, and India-specific HCM
  needs defined.
- **Technical architecture drafted** — data model and the identity-graph relationship
  with [Qeet ID](/id). See [Planned architecture](/people/planned-architecture).

## Next

- First backend skeleton: directory + employment records, grounded on the Qeet ID
  identity graph. Tracked on the [roadmap](/people/roadmap).
