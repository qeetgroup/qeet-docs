Qeet ID implements **SCIM 2.0** for both **Users and Groups**, including **PatchOp**
membership sync — the Okta/Entra-style protocol that keeps directories in step. IdPs
authenticate with a **per-tenant bearer token** distinct from user JWTs.

<Callout type="tip" title="Users AND Groups">
  SCIM Users alone is common; SCIM <em>Groups</em> with PatchOp membership is the part
  that drives [group-level RBAC](/docs/authorization#group-level-roles). Both are
  implemented.
</Callout>

## Per-tenant SCIM token

Provision a token in the tenant, set it in your IdP's SCIM connector, and rotate as
needed. The IdP sends it as `Authorization: Bearer <scim-token>`.

<APIEndpoint method="GET" path="/v1/tenants/{tenantID}/scim" title="SCIM config / status" />
<APIEndpoint method="POST" path="/v1/tenants/{tenantID}/scim/token" title="Rotate the SCIM token" />
<APIEndpoint method="DELETE" path="/v1/tenants/{tenantID}/scim/token" title="Revoke the SCIM token" />

## Users

Full CRUD and filtering; bodies use `application/scim+json`.

<APIEndpoint method="GET" path="/scim/v2/Users" title="List / filter users" />
<APIEndpoint method="POST" path="/scim/v2/Users" title="Provision a user" />
<APIEndpoint method="PATCH" path="/scim/v2/Users/{id}" title="Patch a user (e.g. deactivate)" />

```bash
curl https://api.qeetid.com/scim/v2/Users \
  -H "Authorization: Bearer $SCIM_TOKEN" \
  -H 'Accept: application/scim+json'
```

Deprovisioning (setting `active: false`) disables the account — instant offboarding.

## Groups

Groups support full CRUD plus **PatchOp** membership add/remove — how an IdP keeps
group membership in sync without resending the full group.

<APIEndpoint method="GET" path="/scim/v2/Groups" title="List groups" />
<APIEndpoint method="POST" path="/scim/v2/Groups" title="Create a group" />
<APIEndpoint method="PATCH" path="/scim/v2/Groups/{id}" title="PatchOp — add/remove members" />

```json title="PatchOp — add a member"
{
  "schemas": ["urn:ietf:params:scim:api:messages:2.0:PatchOp"],
  "Operations": [
    { "op": "add", "path": "members", "value": [{ "value": "<scim-user-id>" }] }
  ]
}
```

<Callout type="info">
  Synced groups map to Qeet ID groups, so assigning a role to a group
  ([Authorization → Group roles](/docs/authorization#group-level-roles)) means
  directory membership changes flow straight through to access.
</Callout>
