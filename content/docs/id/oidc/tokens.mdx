After [Authorization Code + PKCE](/id/oidc/authorization-code), the token endpoint
returns an OAuth token response. This page covers the token shape, userinfo, refresh
rotation, introspection, and revocation.

## Token response

```json title="OAuthTokenResponse"
{
  "access_token": "eyJhbGciOiJFUzI1Ni…",
  "id_token": "eyJhbGciOiJFUzI1Ni…",
  "token_type": "Bearer",
  "expires_in": 900,
  "refresh_token": "…",
  "scope": "openid profile email"
}
```

- **`id_token`** — an OIDC ID token signed with **ES256**. Verify it against the
  [JWKS](/id/oidc#discovery--keys).
- **`access_token`** — present it as `Authorization: Bearer <token>` to userinfo and
  your resource servers.

## Userinfo

<APIEndpoint
  method="GET"
  path="/v1/oauth/userinfo"
  title="OIDC userinfo claims"
/>

```bash
curl https://api.id.qeet.in/v1/oauth/userinfo \
  -H "Authorization: Bearer $ACCESS_TOKEN"
# → { "sub": "…", "tenant_id": "…", … }
```

## Refresh-token rotation

Exchange a refresh token for a fresh pair at the same token endpoint. Refresh tokens
**rotate** — the old one is invalidated, and replaying a rotated token revokes the
chain (see [Sessions → theft detection](/id/sessions#refresh-token-rotation--theft-detection)).

<APIEndpoint
  method="POST"
  path="/v1/oauth/token-code"
  title="grant_type=refresh_token"
/>

```bash
curl -X POST https://api.id.qeet.in/v1/oauth/token-code \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d grant_type=refresh_token \
  -d refresh_token=$REFRESH \
  -d client_id=$CLIENT_ID
```

## Introspection (RFC 7662)

Resource servers can ask whether a token is active and get its metadata.

<APIEndpoint
  method="POST"
  path="/oauth/introspect"
  title="Introspect a token"
/>

```bash
curl -X POST https://api.id.qeet.in/oauth/introspect \
  -u "$CLIENT_ID:$CLIENT_SECRET" \
  -d token=$ACCESS_TOKEN
# → { "active": true, "sub": "…", "scope": "…", "exp": … }
```

## Revocation (RFC 7009)

Revoke an access or refresh token explicitly (e.g. on logout from a specific device).

<APIEndpoint method="POST" path="/oauth/revoke" title="Revoke a token" />

```bash
curl -X POST https://api.id.qeet.in/oauth/revoke \
  -u "$CLIENT_ID:$CLIENT_SECRET" \
  -d token=$REFRESH
```

<Callout type="info">
  Introspection and revocation use the same client authentication as the token
  endpoint (form params or HTTP Basic). For full session sign-out across the
  IdP, use [RP-initiated logout](/id/oidc/logout).
</Callout>
