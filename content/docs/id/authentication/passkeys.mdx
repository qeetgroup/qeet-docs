A **passkey** is a WebAuthn credential bound to your origin and stored on the user's
device or synced credential manager. Qeet ID implements passkeys with
[`go-webauthn`](https://github.com/go-webauthn/webauthn): passwordless **registration**
and **login**, plus credential management.

<Callout type="tip" title="This is fully implemented">
  Earlier docs claimed passkeys returned <code>501 Not Implemented</code>. That
  is no longer true — registration and passwordless login both work today.
</Callout>

## How it works

WebAuthn is a two-step ceremony. The server issues a challenge (`begin`), the browser
signs it with the authenticator, and the server verifies the response (`finish`).

<Steps>

<Step>
### Register a passkey (authenticated user)

The user must be signed in. `register/begin` returns
`publicKeyCredentialCreationOptions`; pass them to
`navigator.credentials.create()` and post the result to `register/finish`.

<APIEndpoint method="POST" path="/v1/passkeys/register/begin" title="Get creation options" />
<APIEndpoint method="POST" path="/v1/passkeys/register/finish" title="Store the credential" />
</Step>

<Step>
### Sign in with a passkey (passwordless)

These endpoints are unauthenticated — they _are_ the authentication. `login/begin`
returns `publicKeyCredentialRequestOptions`; pass them to
`navigator.credentials.get()` and post the assertion to `login/finish`, which returns
a token pair.

<APIEndpoint method="POST" path="/v1/passkeys/login/begin" title="Get request options" />
<APIEndpoint method="POST" path="/v1/passkeys/login/finish" title="Assert → token pair" />
</Step>

</Steps>

## Browser ceremony

```ts title="register (browser)"
// 1. Ask Qeet ID for creation options
const options = await fetch("/api/passkeys/register/begin", {
  method: "POST",
}).then((r) => r.json());

// 2. Create the credential with the platform authenticator
const cred = await navigator.credentials.create({ publicKey: decode(options) });

// 3. Send the attestation back to finish registration
await fetch("/api/passkeys/register/finish", {
  method: "POST",
  body: encode(cred),
});
```

```ts title="login (browser)"
const options = await fetch("/api/passkeys/login/begin", {
  method: "POST",
}).then((r) => r.json());
const assertion = await navigator.credentials.get({
  publicKey: decode(options),
});
const tokens = await fetch("/api/passkeys/login/finish", {
  method: "POST",
  body: encode(assertion),
}).then((r) => r.json());
// tokens = TokenPair
```

<Callout type="warn" title="HTTPS required">
  WebAuthn requires a secure context. Passkeys silently fail on plain HTTP
  origins; `localhost` is exempt for development.
</Callout>

## Manage credentials

Users can list and revoke their registered passkeys.

## Passkeys as a second factor

The same registered passkeys can be reused as a **WebAuthn second factor** during
step-up MFA — see [MFA → WebAuthn as a second factor](/id/mfa#webauthn-as-a-second-factor).
