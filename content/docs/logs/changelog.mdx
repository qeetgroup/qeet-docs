Qeet Logs is **in design**. There's no public API yet; this page will track shipped
milestones once development begins.

<Callout type="info" title="In design">
  Qeet Logs is documented from its product requirements — the repo has no code
  yet. See [Concepts](/logs/concepts) for the planned model and the
  [roadmap](/logs/roadmap) for the phased plan. Until it ships, [Qeet ID's
  tamper-evident audit](/id) covers identity-event logging.
</Callout>

## Design phase

- **Product requirements complete** — ingest, query, identity-aware attribution,
  tenant isolation, and data-residency requirements defined.
- **Technical architecture drafted** — ClickHouse-backed storage and the ingest/query
  split. See [Concepts](/logs/concepts).
